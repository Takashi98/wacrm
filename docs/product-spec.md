# Product Spec

## Product Name
WACRM

## Product Summary
WACRM is a WhatsApp-first CRM for Indian small and medium businesses.
It helps businesses manage WhatsApp conversations, convert chats into leads, track lead stages, automate follow-ups, send payment links, and measure results in one place.

## Target Users
- Instagram sellers
- Local shop owners
- Freelancers and consultants
- Coaching centers
- Small D2C brands

## Core Problems
- Leads get lost in WhatsApp chats
- No proper follow-up system
- No clear sales pipeline
- No automation
- No simple way to track payments and conversions

## MVP Goal
Build a simple and usable product that allows a business to:
- see WhatsApp conversations in one inbox
- convert chats into leads
- manage leads in a pipeline
- add notes and tags
- set one simple automation flow
- send Razorpay payment links
- see basic analytics

## Platforms
WACRM must work properly on:
- Mobile
- Desktop

The product should be designed mobile-first, but must have clean responsive layouts for both mobile and desktop users.

## Main Product Areas
1. Inbox
2. Leads
3. Automations
4. Payments
5. Analytics
6. Settings

## Main Pages
- Landing Page
- Features Page
- Pricing Page
- Demo Page
- Login Page
- Signup Page
- Onboarding Flow
- Dashboard Inbox
- Dashboard Leads
- Dashboard Automations
- Dashboard Analytics
- Dashboard Settings
- Billing Page

## MVP Features

### 1. Inbox
- Show conversations
- Show chat messages
- Show message history
- Assign conversation to team member
- Show lead/contact info beside conversation

### 2. Leads
- Create lead from chat
- Lead pipeline stages:
  - New
  - Contacted
  - Qualified
  - Won
  - Lost
- Add tags
- Add notes
- Assign owner

### 3. Automations
- Rule-based automations
- Example:
  - if no reply in X hours → send follow-up
  - if lead tagged hot → notify owner

### 4. Payments
- Create Razorpay payment link
- Send payment link in chat
- Track payment status

### 5. Analytics
- Total leads
- Won leads
- Conversion rate
- Revenue

## MVP Out of Scope
Do not build these in first version:
- advanced drag-drop workflow builder
- AI features
- deep permissions system
- multi-channel inbox
- advanced reports
- ERP or inventory features

## UX Goals
- Very simple UI
- Easy for non-technical users
- Low learning curve
- Clean SaaS design
- Clear call-to-actions
- Helpful empty states
- Proper responsive experience on both mobile and desktop
