# UI Rules

## Product feel
WACRM should feel like a clean, premium, modern SaaS product.
The UI must feel simple, trustworthy, and easy for Indian SMB users.

## Platforms
Design for both mobile and desktop.
Use mobile-first structure, but ensure desktop layouts feel complete and not stretched mobile screens.

## Design principles
- very clear hierarchy
- low clutter
- easy to scan
- simple navigation
- clear CTAs
- generous spacing
- soft rounded cards
- premium but practical look
- no over-designed gradients or flashy visuals

## Color direction
- neutral, light SaaS base
- white / off-white surfaces
- dark text
- green accents only where useful
- muted borders
- subtle shadows

## Typography
- clean sans-serif
- strong section headings
- readable body text
- clear labels and small helper text

## Components
Keep consistent styling for:
- buttons
- inputs
- cards
- badges
- tabs
- sidebars
- drawers
- tables
- empty states

## Empty states
Every main page should have helpful empty states with:
- short explanation
- what the user should do next
- one clear CTA

## Inbox design direction
The Inbox should feel like a mix of:
- WhatsApp Web familiarity
- clean CRM structure
- premium SaaS polish

Desktop layout:
- left conversation list
- center chat area
- right lead/contact sidebar

Mobile layout:
- conversation list screen first
- active chat screen second
- lead details in drawer or stacked panel

## What to avoid
- crowded dashboards
- too many colors
- heavy gradients
- tiny text
- overly enterprise-looking complexity
- confusing hidden actions
