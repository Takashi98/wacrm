# Task 004 - Automation Shell

## Goal
Build the Automations page shell for WACRM.

## Scope
This task includes:
- create a routed automations page
- show a page header
- show a list/table of automation rules using mock data
- include statuses like active/inactive
- include trigger and action summary text
- include a CTA to create a new automation
- include empty states where useful
- support desktop and mobile layouts

## Must include
- route: /automations
- page header
- mock automation rules
- rule cards or rows
- status badge
- trigger summary
- action summary
- last run or last updated text
- create automation CTA

## Out of scope
- visual flow builder
- backend integration
- rule editing logic
- toggle logic
- execution engine
- real data

## Important rules
- keep it UI shell only
- no product logic yet
- keep design aligned with inbox and leads shell
- make it feel like a premium SaaS operations tool
- support both desktop and mobile

## Done when
- /automations route works
- automation shell is visible
- mock rules are shown
- mobile layout works
- no backend integration is required
