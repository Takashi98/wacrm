# Task 005 - Payments Shell

## Goal
Build the Payments page shell for WACRM.

## Scope
This task includes:
- create a routed payments page
- show a page header
- show a list/table of payment links using mock data
- include payment status badges
- include amount, customer name, lead/business name, created date, and payment status
- include a CTA to create a payment link
- include useful empty states
- support desktop and mobile layouts

## Must include
- route: /payments
- page header
- mock payment records
- status badge
- amount
- customer or lead name
- business/source context
- created date or last updated text
- create payment link CTA

## Out of scope
- Razorpay integration
- backend integration
- create payment logic
- resend logic
- filters logic
- search logic
- real payment tracking

## Important rules
- keep it UI shell only
- no product logic yet
- keep design aligned with inbox, leads, and automations shell
- make it feel like a premium SaaS operations tool
- support both desktop and mobile

## Done when
- /payments route works
- payments shell is visible
- mock data is shown
- mobile layout works
- no backend integration is required
